### 简介

在 `webpack4 不太用处理 polyfill`，默认就给我们加上了，所以我们`产出的包特别大`

在 `webpack5` 就默认移除掉了（基于打包速度考虑），如果要用，就`需要自己配置`了

### polyfill是什么

遇到一些更新新的语法比如 promise，gennerater等等，preset-env 并不能帮我们转换

这个时候就需要使用 polyfill 

```js
// 这个包很大，babel7之后就使用安装 core-js regenerator-runtime 就可以了
$ npm i -S @babel/polyfill  // 不建议使用这个包，太大了
```

```js
$ npm i core-js regenerator-runtime -S 
```

// babel.config.js
// 通过 .browserslistrc 条件筛选，babel 只负责转换，我们还需要使用相应工具，@babel/preset-env（集合）
// 但是 @babel/preset-env 不能完成所有的转换，因此需要 polyfill
module.exports = {
  presets: [
    [
      "@babel/preset-env",
      {
        // 默认是 false：不对当前的打包做 polyfill 的填充
        // usage：根据代码进行填充(使用什么代码了，哪些需要 polyfill)
        // entry：根据 .browserslistrc 进行填充
        useBuiltIns: 'usage',
        corejs: 3 // 2 版本的时候不用写，3 版本的时候需要指定一下
      }
    ]
  ],
};
