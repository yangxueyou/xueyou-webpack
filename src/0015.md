### watch模式

```js
// package.json
// 这个时候，修改代码，自动打包
// 书写代码，界面自动更新
script: {
  "build": "webpack --config test.webpack.js --watch"
}
```

### webpack-dev-server


- 开发模式
  - watch
  - live server
- 上面方式的问题：
  - 所有源代码都会重新编译
  - 每次编译成功后，都需要进行文件读写
  - liver server 是 vscode 下的，我们应该找一个 webpack 生态下类似的东西
  - 不能实现局部刷新


```js
$ npm i -D webpack-dev-server
```

```js
// package.json
script: {
  // webpack 4 这块是需要写全对，webpack 5 这样写就可以了 （serve）
  // 如果是 webpack.config.js 默认文件名，就不需要写 --config 
  // 这个就不会产生dist目录了，直接写在内存中，提高编译速度
  server: "webpack serve --config test.webpack.js" 
}
```
